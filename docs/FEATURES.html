Window & Shell

  • 48px window-level ribbon above all surface tabs
  • Drag-drop RibbonBox widgets — reorder by dragging the grip handle; blue drop indicator

AuxDeck Module — Independent Audio Routing

The AuxDeck Module (com.mcaster1.auxdeck) provides fully independent auxiliary audio players with dual-bus routing: each AuxDeck has its own AIR output and CUE output, each routed to a separate physical audio device via PortAudio. Both buses can output simultaneously, enabling preview-before-broadcast workflows. 26 output devices are enumerated across WASAPI, DirectSound, MME, and ASIO host APIs.

Use Cases

DJ Preview / Cueing

  • Load upcoming tracks on an AuxDeck, route CUE to headphones (separate device), preview before going on-air
  • Toggle CUE off when done, toggle AIR on to send to broadcast
  • Both buses are fully independent — on/off control flanking the Mute button

Interview Playback

  • During a podcast or radio show, use an AuxDeck to play sound bites, clips, or pre-recorded segments
  • Route to a separate audio device for studio monitor while the main decks continue playing music on-air
  • Independent volume and transport controls per AuxDeck

Jingle / Stinger Deck

  • Dedicated AuxDeck for instant jingle playback
  • Route AIR to the broadcast output for immediate on-air insertion
  • Hot cue buttons (H1–H4) for instant jump to pre-set jingle start points

Multi-Room Audio

  • Route different AuxDecks to different physical audio devices
  • Examples: lobby speakers, studio monitors, streaming encoder input
  • Each AuxDeck is independent with its own AIR and CUE device — device collision prevention blocks the same device on both buses

Recording Preview

  • Record on one AuxDeck while previewing the recording on another AuxDeck's CUE output before deciding to broadcast it
  • Per-bus routing: dedicated PortAudio stream when a specific device is selected, global fallback when “(Use Global Default)” is chosen

DeckPlayer & AutoDJ Improvements

The core DeckPlayer and AutoDJ modules received significant layout and reliability improvements in the v0.4.0-beta release cycle.
  • Crossfader top-aligned between Deck A and Deck B in the center column
  • Playlist/AutoDJ permanently embedded below crossfader — auto-created when deck exists
  • AutoDJ auto-plays first track immediately when enabled (500ms delayed trigger)
  • S-curve crossfade corrected — equal-power sin/cos prevents clipping at center
  • Loop boundary guard — prevents infinite loop when loopOut <= loopIn
  • Ring buffer overflow — StreamReader uses int64_t indices for weeks of streaming
  • Rich ICY metadata — DeckA/DeckB emit full artist/title/album/genre/year/BPM
